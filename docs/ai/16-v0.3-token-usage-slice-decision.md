# v0.3 Token Usage Slice Decision

## 1. 背景

v0.3/P2 的主要剩余产品缺口是 Token 花费统计面板。当前 AI 调用已经通过 OpenAI-compatible `/chat/completions` 完成，但客户端只读取回复文本，没有读取或持久化响应中的 `usage`。

用户已确认响应式适配正常，因此可以推进 Token 统计。

## 2. 本切片选择

本切片选择“Token usage 记录与基础统计面板”，暂不做真实花费金额计算。

原因：

- OpenAI-compatible 响应通常返回 `usage.prompt_tokens`、`usage.completion_tokens`、`usage.total_tokens`。
- 先记录模型实际返回的 usage 数据，能为后续价格计算打基础。
- 不同供应商、模型、时间区间和输入/输出价格会变化，直接做金额统计需要价格配置和版本化，复杂度较高。
- 第一切片可以不改变聊天主路径，只在成功 AI 回复后补充记录 usage。

## 3. 当前目标

用户可以在页面中看到自己账号下当天和最近一段时间的 Token 使用量，包括 prompt、completion、total，并可按模型/API 配置维度查看基础汇总。

## 4. 产出

- 扩展 AI reply 结果结构，携带 prompt/completion/total token usage。
- 解析 OpenAI-compatible 响应中的 `usage` 字段。
- 新增 Token usage 持久化表。
- 成功保存 AI 回复后记录对应 usage。
- 新增基础统计入口和页面。
- 页面展示当天和最近 7 天的 usage 汇总。

## 5. 内容边界

- 本切片只统计 Token 数量。
- 不计算真实金额。
- 不做价格配置。
- 不做供应商价格同步。
- 不做图表库。
- 不追溯历史消息 usage。
- 不改变消息内容、异步发送、轮询、AI 互评或主题引导行为。
- 如果模型响应没有 `usage`，不阻断聊天回复，只记录为无 usage。

## 6. 验收检查

- AI 客户端能解析 `usage` 字段。
- 成功 AI 回复后能持久化 usage。
- 没有 usage 的模型响应不影响回复保存。
- 用户可以访问 Token 统计页面。
- 页面显示今天和最近 7 天的 prompt/completion/total token 汇总。
- 自动测试和构建通过。

## 7. 明确不做

- 不做金额计算。
- 不做按价格版本回溯计费。
- 不做实时账单或余额。
- 不做成本告警。
- 不做跨供应商价格管理。
