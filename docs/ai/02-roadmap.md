# Product Roadmap

## 1. v0.1 MVP

### 目标

验证“网页端 AI 群聊”这个核心产品是否成立：用户能创建一个群聊，配置多个 AI 角色，并看到不同 AI 角色围绕同一条用户消息回复。

### 包含

- AI 群聊房间。
- 群聊名称。
- AI 角色添加与基础配置。
- 至少两个 AI 角色参与同一轮讨论。
- 用户发送消息，AI 角色生成回复。
- 清晰显示发送者身份。
- 基础模型 API 配置。
- 群聊消息保存与再次查看。
- 网页端可用界面。

### 不包含

- 用户登录与多用户账号隔离。
- 完整 API 管理后台。
- AI 角色删除、编辑和发言权限开关。
- 稳定实时消息推送体验。
- AI 角色之间的补充或反驳。
- 群聊主题引导。
- 思考强度配置。
- Token 花费统计面板。
- 文件上传与 AI 文件分析。
- 工具调用。
- 白天和黑夜模式。
- 公开社区或多人实时协作聊天室。

### 完成标准

- 用户可以在网页端创建一个带名称的 AI 群聊。
- 用户可以添加至少两个 AI 角色，并配置角色名称、人设、回复风格和模型。
- 用户可以发送一条消息，并看到至少两个 AI 角色分别回复。
- 每条消息都能清楚显示发送者身份。
- 群聊消息能保存，用户再次进入同一群聊时能看到历史消息。
- 用户可以完成基础模型 API 配置，并让 AI 角色使用该配置产生回复。

## 2. v0.2 可用版

### 目标

把 MVP 从“能验证核心闭环”推进到“可以持续使用”，补齐账号、管理、实时体验和角色控制能力。

### 包含

- 用户登录与账号数据隔离。
- 完整 API 管理，包括多个 API、模型和账号绑定关系。
- AI 角色删除、编辑和发言权限开关。
- 稳定实时消息推送体验。
- AI 角色之间的补充或反驳。
- MVP 级验收与基础测试。
- 技术栈和项目结构选择。

### 不包含

- 文件上传与 AI 文件分析。
- 工具调用。
- 插件或角色市场。
- 公开社区或大型多人聊天室。
- 多供应商、多模型高级路由。

### 完成标准

- 不同用户登录后只能看到和管理自己的群聊、AI 角色、API 配置和消息历史。
- 用户可以新增、编辑、删除 AI 角色，并控制单个角色是否允许发言。
- 用户可以在设置中管理多个 API 或模型配置，并在角色配置中选择可用模型。
- 群聊消息能以稳定的实时方式显示，不需要手动刷新页面才能看到 AI 回复。
- AI 角色可以在同一讨论中对其他 AI 角色的回答进行补充或反驳。
- 核心流程有明确验收项，并能通过基础测试或人工验收重复验证。

## 3. v0.3 增强版

### 目标

增强使用体验、成本感知和讨论组织能力，让产品从“可用”走向“更好用”。

### 包含

- 群聊主题引导。
- 思考强度配置。
- Token 花费统计面板。
- 白天和黑夜模式。
- 更好的桌面端与移动端适配。
- 群聊历史检索、筛选和管理。
- 开发者设置文档。
- 配置和环境管理。

### 不包含

- 文件上传与 AI 文件分析。
- 工具调用。
- 插件或角色市场。
- 公开社区或大型多人聊天室。

### 完成标准

- 用户可以为群聊设置讨论主题，AI 回复能围绕主题保持基本一致性。
- 用户可以为模型或角色设置思考强度，并能在实际回复中使用该配置。
- 用户可以查看当天或指定范围内的 Token 使用与花费统计。
- 用户可以在白天模式和黑夜模式之间切换。
- 核心页面在桌面端可稳定使用，移动端至少能完成主要阅读和发送流程。
- 用户可以检索或筛选群聊历史消息。
- 新开发者可以根据文档完成本地配置和运行准备。

## 4. v1.0 稳定版

### 目标

形成稳定、可维护、可部署的正式版本，并在不偏离 AI 群聊核心价值的前提下加入高价值远期能力。

### 包含

- 文件上传与 AI 文件分析。
- 工具调用能力。
- 多供应商、多模型高级路由。
- 部署文档。
- 可观测性、日志和错误报告策略。
- CI 检查。
- 核心 UX、可访问性和性能优化。

### 不包含

- 通用社交 IM。
- 公开大型聊天室。
- 模型训练、微调或模型托管。
- 默认开放的无限制命令执行。
- 不受审核或治理的公开插件市场。

### 完成标准

- 用户可以上传受支持的文件，并让 AI 在群聊中基于文件内容进行分析。
- 用户可以在受控范围内使用工具调用，且工具权限、执行结果和失败状态清晰可见。
- 系统可以接入多个模型供应商，并能在角色或配置层面选择合适模型。
- 项目具备清晰部署文档，能够按文档部署一个稳定环境。
- 关键错误、模型调用失败和消息处理异常有日志或错误报告可追踪。
- CI 能检查核心质量门槛，避免明显破坏主流程的变更进入稳定版本。
- 产品仍保持 AI 多角色群聊定位，没有扩张成通用社交或模型平台。
