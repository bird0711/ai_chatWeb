# QA Status

## Current QA Position

The repository is no longer in a one-time MVP closeout state. QA now follows the current engineering workflow and is tied to command-based verification plus targeted manual smoke checks.

## Automated Verification Baseline

Current baseline commands:

- `make check`
- `make integration`

These cover:

- default Go test and lint quality gate
- real MySQL verification
- real Redis verification
- main HTTP flow verification over real dependencies

## Manual Verification Baseline

Use `docs/ai/release-checklist.md` for the current manual smoke path.

Minimum manual checks:

1. open `/login`
2. log in or register
3. confirm `/health`
4. create a chat
5. add roles
6. send a message
7. upload a supported file
8. confirm usage/statistics page loads

## Current Rule

Do not treat a branch as handoff-ready only because unit tests passed. The current baseline expects both:

1. automated command verification
2. a short manual smoke pass
