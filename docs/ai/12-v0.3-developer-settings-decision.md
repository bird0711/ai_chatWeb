# v0.3 Developer Settings Slice Decision

## 1. 背景

v0.3/P2 还剩开发者设置文档、配置和环境管理、历史检索、思考强度、Token 统计、更广泛适配等项目。

用户已确认白天/黑夜模式功能正常，因此可以继续推进一个低风险小切片。

## 2. 本切片选择

本切片选择“开发者设置文档”和基础“配置和环境管理”说明。

原因：

- 属于 v0.3/P2 明确范围。
- 不改业务代码，不影响 AI 调用、消息存储、账号隔离或异步回复。
- 能降低后续本地运行、排错和交接成本。
- 当前项目已经多次遇到 MySQL/Redis、本地端口、模型 API 配置和上传目录相关问题，文档化收益明确。

## 3. 当前目标

让新开发者可以根据文档完成本地环境准备、配置服务、启动项目、配置模型 API，并知道常见故障如何定位。

## 4. 产出

- 新增 `.env.example`。
- 新增 `docs/ai/12-v0.3-developer-settings-decision.md`。
- 新增 `docs/ai/developer-settings.md`。
- 更新 `README.md` 指向开发者设置文档。

## 5. 内容边界

- 记录所有当前支持的环境变量。
- 说明默认 MySQL、Redis、HTTP 端口和上传目录。
- 说明本地启动步骤。
- 说明模型 API 配置入口和 OpenAI-compatible 要求。
- 说明常见问题排查。
- 不新增部署文档。
- 不新增 CI。
- 不新增容器编排。
- 不改变运行时代码。

## 6. 验收检查

- `.env.example` 包含项目当前支持的主要环境变量。
- 开发者文档说明 MySQL、Redis、启动、上传目录、模型 API、主题模式和常见故障。
- `README.md` 指向开发者设置文档。
- 自动测试和构建仍通过。

## 7. 明确不做

- 不做生产部署文档。
- 不做 Docker Compose。
- 不做 CI 配置。
- 不做日志/可观测性专项。
- 不做安全加固专项。

## 8. 下一轮验证切片

验证文档是否足以让用户按本地环境启动项目，并确认 `.env.example` 与当前代码支持的环境变量一致。
