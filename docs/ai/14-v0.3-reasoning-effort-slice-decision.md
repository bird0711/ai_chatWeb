# v0.3 Reasoning Effort Slice Decision

## 1. 背景

v0.3/P2 仍包含思考强度配置、Token 花费统计、更好的桌面端与移动端适配等能力。

用户已确认当前群聊历史检索和筛选功能正常，因此可以推进下一个 P2 小切片。

## 2. 本切片选择

本切片选择“角色级思考强度配置”。

原因：

- 属于 v0.3/P2 明确范围。
- 与现有 AI 角色创建、编辑和模型 API 调用路径天然关联。
- 可以做成轻量字段，不改变消息存储或异步回复架构。
- OpenAI-compatible 请求可在兼容模型中透传 `reasoning_effort`，不兼容模型由供应商返回错误，用户可切回默认。

## 3. 当前目标

用户在创建或编辑 AI 角色时，可以选择默认、低、中、高思考强度。默认表示不向模型 API 传递额外参数；低、中、高会在聊天回复和 AI 互评调用中传递。

## 4. 产出

- 为 AI 角色新增 `reasoning_effort` 字段。
- 为 MySQL `roles` 表新增兼容迁移。
- 在角色创建和编辑表单中新增思考强度选择。
- 在角色卡片中显示当前思考强度。
- 在 OpenAI-compatible `/chat/completions` 请求体中按角色配置透传 `reasoning_effort`。
- 添加服务层、HTTP 模板和 AI 客户端请求体测试。

## 5. 内容边界

- 仅支持角色级配置。
- 仅支持默认、低、中、高。
- 默认值为空，不传递 `reasoning_effort`。
- 不做模型能力自动探测。
- 不做供应商专用参数映射。
- 不做 Token 统计或成本展示。
- 不改变消息存储、异步发送、轮询、主题引导、AI 互评逻辑。

## 6. 验收检查

- 创建角色时可以选择思考强度。
- 编辑角色时可以修改思考强度。
- 角色卡片显示当前思考强度。
- 默认思考强度不会向模型请求体传递 `reasoning_effort`。
- 低、中、高会向模型请求体传递对应值。
- 自动测试和构建通过。

## 7. 明确不做

- 不做 Token 花费统计。
- 不做模型供应商能力矩阵。
- 不做高级推理参数面板。
- 不做按群聊统一覆盖角色配置。

## 8. 下一轮验证切片

由用户在本机浏览器验证创建/编辑角色的思考强度配置是否正常，并确认聊天回复主路径没有回归。
