# v0.2 Second Slice Task Decision

## 1. 当前目标

在 v0.2 第一切片已验证的基础上，补齐 AI 角色的基础生命周期管理：编辑角色配置，以及开启或关闭角色发言权限。

## 2. 本切片选择

本切片实现：

- AI 角色编辑。
- AI 角色发言权限开关。

暂不实现：

- 用户登录与账号隔离。
- 完整多 API 管理。
- AI 角色之间的补充或反驳。

## 3. 选择理由

- 用户已经可以添加和删除 AI 角色，编辑和发言权限开关是同一条角色管理链路上的最小补齐。
- 关闭发言权限可以解决“暂时不想让某个角色参与讨论，但不想删除历史配置”的常见场景。
- 该切片可以复用现有角色表单、模型下拉和发送消息规则，不需要重构实时消息方案。
- 登录、完整 API 管理和 AI-to-AI 互动都比本切片更大，不适合作为当前下一步。

## 4. UI 行为

- 群聊详情页的每个 AI 角色卡片提供“编辑”入口。
- 编辑表单允许修改：
  - 角色名称。
  - 人设。
  - 回复风格。
  - 模型。
  - 是否允许发言。
- 模型仍只能从模型设置中已保存的支持模型列表选择。
- 角色卡片显示当前发言状态：
  - 允许发言。
  - 已禁言。
- 角色卡片提供一个快速切换发言权限的按钮。
- 保存编辑后回到当前群聊详情页。
- 保存失败时在当前页面显示明确错误。

## 5. 后端行为

- `roles` 数据结构增加发言权限字段。
- 迁移兼容已有数据，已有角色默认允许发言。
- 新增角色时默认允许发言。
- 编辑角色时校验：
  - 角色必须属于当前群聊。
  - 名称、人设、回复风格、模型不能为空。
  - 模型必须存在于当前模型配置的支持模型列表中。
- 发送消息时只让允许发言的 AI 角色参与回复。
- 发送消息前至少需要两个允许发言的 AI 角色；否则阻止发送并提示。
- 删除角色仍保留历史消息。

## 6. 验收检查

- 用户可以编辑已有 AI 角色的名称、人设、回复风格和模型。
- 编辑后刷新页面，新配置仍然保存。
- 用户可以关闭某个 AI 角色的发言权限。
- 关闭发言权限后，该角色不再参与新消息回复。
- 用户可以重新开启某个 AI 角色的发言权限。
- 重新开启后，该角色可以参与后续回复。
- 当允许发言的 AI 角色少于两个时，发送消息被阻止，并显示明确提示。
- 模型选择仍来自模型设置中的支持模型列表，不能手动输入未配置模型。
- v0.2 第一切片不回退：异步发送、AI 回复自动追加、角色删除、群聊删除仍可用。
- 自动测试覆盖服务层规则和 HTTP 路由行为。

## 7. 明确不做

- 不做用户登录与账号隔离。
- 不做完整多 API 管理后台。
- 不做 AI 角色互相补充或反驳。
- 不做群聊主题引导。
- 不做思考强度配置。
- 不做 Token 花费统计。
- 不做文件上传。
- 不做工具调用。
- 不做白天和黑夜模式。
- 不做公开社区或多人实时协作聊天室。

## 8. 停止条件

- 如果需要重构账号体系才能完成本切片，停止说明。
- 如果需要改变现有模型配置方式，停止说明。
- 如果实现会破坏 v0.2 第一切片的异步聊天体验，停止说明。
- 如果用户把范围扩展到登录、完整 API 管理、AI-to-AI 互动或 P2/P3 功能，先写入 BACKLOG，不在本切片实现。
