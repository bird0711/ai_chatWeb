# v0.3 Theme Mode Slice Decision

## 1. 背景

v0.3 已完成聊天化消息展示、聊天详情页布局、AI 头像、本地头像上传、Enter 发送兼容性修复和群聊主题引导。

用户确认主题引导功能正常后，可以继续推进下一个 P2 小切片。

## 2. 本切片选择

本切片选择“白天和黑夜模式”。

原因：

- 属于 v0.3/P2 明确范围。
- 范围主要集中在前端模板、CSS 和少量 JS。
- 不需要改变后端业务、消息存储、AI 调用、账号隔离或 API 管理。
- 能继续改善用户长时间聊天阅读体验。

## 3. 当前目标

允许用户在主要页面切换白天模式和黑夜模式，并在同一浏览器中保留选择。

## 4. UI 行为

- 已登录后的主要页面导航栏展示主题切换按钮。
- 默认跟随系统偏好。
- 用户点击按钮后在白天/黑夜之间切换。
- 选择保存在 `localStorage`。
- 聊天详情页、群聊列表、模型设置、系统状态和错误页都加载主题脚本。

## 5. 技术行为

- 新增 `web/static/theme.js`。
- 使用 `document.documentElement[data-theme]` 标记当前主题。
- 使用 `localStorage` 保存 `ai-chat-theme`。
- CSS 使用 `[data-theme="dark"]` 做暗色覆盖。
- 不新增后端接口。
- 不新增数据库字段。

## 6. 验收检查

- 导航栏能看到“黑夜模式/白天模式”切换按钮。
- 点击按钮后页面立即切换主题。
- 刷新页面后保留上次选择。
- 聊天详情页消息、角色侧栏、设置侧栏、输入框和按钮在暗色模式下可读。
- 群聊列表、模型设置和系统状态页在暗色模式下可读。
- 现有登录、账号隔离、API 管理、角色生命周期、异步回复、AI 互评、主题引导、头像和 Enter 发送不回退。

## 7. 明确不做

- 不做多套主题。
- 不做服务端账号级主题偏好。
- 不做完整设计系统重构。
- 不做可访问性全面审计。
- 不做性能优化专项。

## 8. 停止条件

- 如果主题模式需要后端持久化，停止并收窄。
- 如果主题模式导致聊天主路径不可用，优先修复主路径。
- 如果实现扩大到设计系统或多主题配置，停止并收窄。

## 9. 下一轮验证切片

验证 `docs/ai/11-v0.3-theme-mode-slice-decision.md`：

1. 在群聊列表页切换黑夜模式。
2. 刷新页面确认主题保留。
3. 打开群聊详情页确认聊天区和左右侧栏可读。
4. 切回白天模式并刷新确认保留。
5. 回归检查主题引导、Enter 发送、AI 回复和 AI 互评。
