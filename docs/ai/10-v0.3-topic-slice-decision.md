# v0.3 Topic Guidance Slice Decision

## 1. 背景

v0.2 P1 已完成并通过用户本机验证。v0.3 已完成聊天详情页消息展示、布局、头像、视觉 refinement 和 Enter 发送快捷键。

用户已确认 Enter 发送快捷键正常，因此当前可以继续选择下一个 P2 小切片。

## 2. 本切片选择

本切片选择“群聊主题引导”。

原因：

- 属于 v0.3/P2 明确范围。
- 能直接改善 AI 群聊讨论聚焦度。
- 可以基于现有 chat、prompt、HTTP/UI 和 MySQL 迁移完成。
- 不需要引入 Token 统计、文件上传、工具调用、WebSocket 或多供应商高级路由。

## 3. 当前目标

允许用户为单个群聊设置一个可选讨论主题，并让 AI 普通回复与 AI 互评优先围绕该主题展开。

## 4. UI 行为

- 群聊详情页显示当前主题摘要。
- 右侧设置区增加“讨论主题”折叠菜单。
- 用户可以保存或清空主题。
- 主题最长 500 个字符。
- 主题保存后返回当前群聊详情页。

## 5. 后端行为

- `chats` 增加 `topic` 字段。
- 主题按账号隔离，用户只能修改自己的群聊主题。
- AI 普通回复系统提示词包含群聊主题。
- AI 互评系统提示词包含群聊主题。
- 不改变消息存储结构。
- 不改变异步发送、轮询、AI 互评、角色权限、账号隔离和 API 管理主链路。

## 6. 验收检查

- 群聊详情页能看到“讨论主题”入口。
- 保存主题后页面展示“主题：...”。
- 清空主题后不再展示主题摘要。
- 超过 500 字符的主题被拒绝。
- AI 普通回复 prompt 包含主题约束。
- AI 互评 prompt 包含主题约束。
- 现有登录、账号隔离、API 管理、角色生命周期、异步回复、AI 互评、头像和 Enter 发送不回退。

## 7. 明确不做

- 不做主题模板库。
- 不做自动生成主题。
- 不做多主题或主题历史。
- 不做 Token 统计。
- 不做文件上传。
- 不做工具调用。
- 不做 WebSocket 重构。
- 不做完整设计系统重构。

## 8. 停止条件

- 如果主题引导需要改变消息存储结构，停止并说明。
- 如果主题引导需要重写 AI 异步回复链路，停止并收窄。
- 如果实现扩大到 Token 统计、文件、工具或多供应商高级路由，停止并收窄。

## 9. 下一轮验证切片

验证 `docs/ai/10-v0.3-topic-slice-decision.md`：

1. 在群聊详情页保存一个主题。
2. 确认页面展示主题摘要。
3. 清空主题并确认摘要消失。
4. 发送消息并确认普通 AI 回复仍正常追加。
5. 开启 AI 互评并确认互评回复仍正常追加。
6. 检查登录、账号隔离、API 管理、角色生命周期、头像和 Enter 发送主路径不回退。
